# 影片錄製指南

## 錄製前準備

### 環境設定清單

#### 桌面準備
- [ ] 清理桌面，只保留必要圖示
- [ ] 關閉所有不必要的應用程式
- [ ] 關閉通知（系統、Slack、郵件等）
- [ ] 設定統一的桌布（建議純色或工作坊品牌背景）
- [ ] 準備好所有需要的分頁和視窗

#### 瀏覽器設定
- [ ] 使用無痕/隱私模式避免個人資料露出
- [ ] 安裝深色主題保護眼睛
- [ ] 調整適當的縮放比例（建議 125-150%）
- [ ] 關閉自動填入和密碼管理器
- [ ] 準備好書籤列的重要連結

#### 開發環境
- [ ] VS Code 主題統一（建議 Dark+ 或 One Dark Pro）
- [ ] 字體大小調整為易讀（建議 16-18px）
- [ ] 關閉 minimap 獲得更多空間
- [ ] 安裝語法高亮擴充功能
- [ ] 設定自動儲存避免手動操作

## 音訊錄製技巧

### 麥克風設定

#### 硬體建議
```markdown
入門級：
- 耳機麥克風（避免回音）
- USB 麥克風（如 Blue Yeti Nano）

進階級：
- 專業 USB 麥克風（如 Blue Yeti、Audio-Technica AT2020）
- XLR 麥克風 + 音訊介面

環境準備：
- 選擇安靜的錄製空間
- 使用防噴罩減少爆音
- 軟質材料吸音（毛毯、泡棉）
```

#### 軟體設定
```json
{

  },
```

### 旁白技巧

#### 聲音表現
1. **語速控制**
   - 正常速度：150-160 字/分鐘
   - Demo 時稍慢：130-140 字/分鐘
   - 保持穩定節奏

2. **語調變化**
   - 重點處提高音調
   - 解釋時降低音調
   - 驚喜處加入情緒

3. **清晰度**
   - 咬字清楚
   - 避免口語贅詞
   - 技術名詞放慢速度

#### 錄製流程
```markdown
準備階段：
1. 喝溫水潤喉
2. 做發聲練習
3. 測試錄音音量
4. 準備腳本或大綱

錄製中：
1. 保持坐姿端正
2. 麥克風距離 15-20 公分
3. 看著螢幕自然說話
4. 錯誤時停頓後重說

後製理：
1. 剪除長停頓
2. 調整音量一致性
3. 加入背景音樂（-20dB）
```

## 螢幕錄製最佳實踐

### 錄製區域設定

#### 解析度選擇
```yaml
4K (3840x2160):
  優點: 最高畫質、後製彈性大
  缺點: 檔案巨大、需要強大硬體
  建議: 專業製作使用

1080p (1920x1080):
  優點: 標準高畫質、普遍支援
  缺點: 4K 螢幕需要縮放
  建議: 一般錄製首選

720p (1280x720):
  優點: 檔案較小、串流友善
  缺點: 畫質稍差
  建議: 快速分享或網路不佳時
```

```markdown
自訂區域：
```


```yaml
MP4 (H.264):
  用途: 通用格式、各平台支援
```

## 常見問題解決

### 技術問題

```markdown
Q: 錄製卡頓或掉幀
A: 降低錄製解析度、關閉其他程式、使用硬體加速編碼

Q: 音訊有雜音或回音
A: 使用耳機、調整麥克風位置、開啟降噪功能

Q: 檔案太大無法上傳
A: 使用 HandBrake 壓縮、降低位元率、分段上傳

Q: 影音不同步
A: 重新編碼、使用專業剪輯軟體、檢查源檔案
```

### 內容問題

```markdown
Q: 講解太快聽不清
A: 後製時可局部放慢、重錄該段落、加入更多停頓

Q: 操作步驟不清楚
A: 加入箭頭標註、放大關鍵區域、增加文字說明

Q: 程式碼看不清楚
A: 增大字體、使用高對比主題、局部放大展示

Q: 觀眾注意力分散
A: 縮短單一片段、加入章節分隔、使用視覺提示
```
