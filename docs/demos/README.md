## 使用指南

### 快速開始

1. **選擇應用程式**
   ```bash
   cd sample-app-source/[app-name]/src
   ```

2. **在瀏覽器開啟**
   ```bash
   # 方法 1: 直接開啟
   open index.html
   
   # 方法 2: 使用 HTTP 伺服器
   python -m http.server 8000
   ```

3. **執行測試**
   ```bash
   # 安裝 Playwright
   npm install -D @playwright/test
   
   # 執行測試
   npx playwright test [app-name]/tests/
   ```

### TDD 工作流程

1. **紅燈階段** - 撰寫失敗的測試
   ```javascript
   test('應該能新增項目', async ({ page }) => {
     // 測試邏輯
   });
   ```

2. **綠燈階段** - 實作功能通過測試
   ```javascript
   function addItem(item) {
     // 實作邏輯
   }
   ```

3. **重構階段** - 優化程式碼
   ```javascript
   // 改善程式碼品質，保持測試通過
   ```

## 常見問題

### Q: 為什麼使用 Vanilla JavaScript？
A: 讓學習者專注於核心概念，而非框架語法。理解基礎後可輕鬆遷移到任何框架。

### Q: 如何處理測試中的非同步操作？
A: Playwright 內建優秀的等待機制，使用 `await` 和適當的選擇器即可。

### Q: 可以改用其他測試框架嗎？
A: 可以，但工作坊以 Playwright 為主，因其對 AI 生成測試的支援較好。

### Q: 如何評估測試品質？
A: 不只看覆蓋率，更重要的是測試案例的意義和維護性。
