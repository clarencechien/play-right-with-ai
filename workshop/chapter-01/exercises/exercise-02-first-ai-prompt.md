# 練習 2：第一個 AI 提示

## 目標
練習使用 AI 生成簡單的程式碼，體驗 AI 指揮家的工作方式。

## 預計時間
20-30 分鐘

## 任務：生成一個簡單的計算機應用

### 步驟 1：準備提示詞

使用以下模板創建您的第一個提示詞：

```markdown
# 任務：創建網頁計算機

## 需求
我需要一個簡單的網頁計算機，具有以下功能：
1. 基本運算：加、減、乘、除
2. 清除按鈕（C）和刪除按鈕（←）
3. 小數點支援
4. 響應式設計，適配手機螢幕

## 技術規格
- 純 HTML、CSS、JavaScript（不使用框架）
- 使用 CSS Grid 布局按鈕
- 按鈕要有點擊效果
- 顯示螢幕要顯示當前運算過程

## 樣式要求
- 使用現代化設計
- 深色主題
- 圓角按鈕
- 適當的陰影效果

請生成完整的程式碼，包含詳細的中文註解。
```

### 步驟 2：提交給 AI

將上述提示詞提交給您選擇的 AI 服務（Claude、Gemini 或 GPT-4）。

### 步驟 3：保存並測試

1. 創建資料夾 `my-calculator`
2. 將 AI 生成的程式碼保存為：
   - `index.html`
   - `styles.css`（如果分離）
   - `script.js`（如果分離）

3. 在瀏覽器中開啟測試

### 步驟 4：迭代改進

如果結果不滿意，嘗試以下改進提示：

```markdown
# 改進請求

剛才生成的計算機很好，但我想要以下改進：

1. [具體描述需要改進的地方]
2. [例如：按鈕太小、顏色不夠對比等]

請更新程式碼。
```

## 實作任務

### 任務 A：記錄差異
比較不同 AI 服務的輸出：
1. 程式碼風格差異
2. 註解詳細程度
3. 功能完整性
4. UI 設計美觀度

### 任務 B：優化提示詞
嘗試修改提示詞，觀察結果變化：
1. 加入更多細節
2. 使用不同的描述方式
3. 指定程式碼風格偏好

### 任務 C：除錯練習
故意在提示中加入矛盾需求，觀察 AI 如何處理：
```markdown
創建一個計算機，但是：
- 只能做加法和減法
- 需要支援科學計算功能
```

## 評估標準

### 基本要求（必須完成）
- [ ] 計算機可以正常執行四則運算
- [ ] UI 在手機和桌面都能正常顯示
- [ ] 程式碼包含中文註解

### 進階要求（選擇性）
- [ ] 支援鍵盤輸入
- [ ] 有運算歷史記錄
- [ ] 支援括號運算
- [ ] 有錯誤處理（如除以零）

## 反思問題

完成練習後，請思考並記錄：

1. **提示詞的重要性**
   - 哪些細節對生成結果影響最大？
   - 如何平衡詳細度和靈活性？

2. **AI 的能力邊界**
   - AI 在哪些方面表現優秀？
   - 哪些地方需要人工調整？

3. **迭代的價值**
   - 第一次生成的結果滿意嗎？
   - 經過幾次迭代才達到理想效果？

## 提交作業

請提交以下內容：
1. 最終的計算機程式碼
2. 使用的所有提示詞歷史
3. 對比不同版本的截圖
4. 反思問題的回答（200-300字）

## 延伸挑戰

如果您完成了基本任務，可以嘗試：

1. **進階計算機**
   - 加入科學計算功能
   - 支援單位轉換
   - 加入主題切換

2. **提示詞工程**
   - 創建一個「提示詞模板」
   - 測試極簡提示 vs 詳細提示的差異
   - 研究如何讓 AI 生成特定風格的程式碼

3. **多 AI 協作**
   - 用一個 AI 生成基礎版本
   - 用另一個 AI 進行改進
   - 比較協作效果
