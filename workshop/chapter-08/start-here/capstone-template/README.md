# Capstone 專案：線上學習平台

## 專案需求

你需要使用 AI 工具協助開發一個完整的線上學習平台，包含以下功能：

### 核心功能需求

1. **用戶系統**
   - 註冊/登入/登出
   - 個人資料管理
   - 密碼重設

2. **課程管理**
   - 課程列表顯示
   - 課程詳情頁面
   - 課程搜尋和篩選
   - 課程評分和評論

3. **學習進度**
   - 課程進度追蹤
   - 學習歷史記錄
   - 成就系統

4. **互動功能**
   - 討論區
   - 問答系統
   - 即時通知

### 技術要求

- 前端：React/Vue/Angular（選擇其一）
- 後端：Node.js/Python/Java（選擇其一）
- 資料庫：PostgreSQL/MySQL/MongoDB（選擇其一）
- 測試：Playwright E2E 測試覆蓋率 > 80%

### 品質要求

1. **性能**
   - 頁面載入時間 < 3 秒
   - API 回應時間 < 500ms
   - 支援 100 個並發用戶

2. **安全性**
   - 實作認證和授權
   - 防護 XSS 和 SQL 注入
   - 使用 HTTPS
   - 實作 CSRF 保護

3. **可用性**
   - 響應式設計（支援手機、平板、桌面）
   - 無障礙設計（WCAG 2.1 AA 級）
   - 跨瀏覽器相容

4. **測試**
   - 單元測試覆蓋率 > 70%
   - E2E 測試覆蓋主要用戶流程
   - 性能測試
   - 安全性測試

## 開始步驟

1. **環境設置**
   ```bash
   # 初始化專案
   npm init -y
   
   # 安裝 Playwright
   npm install -D @playwright/test
   
   # 初始化 Playwright
   npx playwright install
   ```

2. **使用 AI 生成基礎架構**
   - 使用提供的提示詞模板
   - 請 AI 幫助設計系統架構
   - 生成初始程式碼框架

3. **迭代開發**
   - 實作一個功能
   - 寫測試
   - 執行測試
   - 修復問題
   - 重複

4. **品質保證**
   - 執行所有測試
   - 性能優化
   - 安全性檢查
   - 程式碼審查

## AI 提示詞範例

### 架構設計提示詞
```
我需要設計一個線上學習平台的系統架構。
要求：
1. 支援 100 個並發用戶
2. 包含用戶系統、課程管理、學習進度追蹤
3. 使用微服務架構
4. 考慮擴展性和維護性

請提供：
1. 系統架構圖
2. 技術棧建議
3. 資料庫設計
4. API 設計原則
```

### 功能實作提示詞
```
請幫我實作課程列表頁面的功能。
需求：
1. 顯示課程卡片（標題、描述、講師、評分）
2. 支援分頁（每頁 12 個課程）
3. 支援搜尋（課程名稱、講師名稱）
4. 支援篩選（類別、難度、評分）
5. 支援排序（最新、最熱門、評分最高）

使用 React 和 TypeScript，請提供完整的元件程式碼。
```

### 測試生成提示詞
```
基於以下用戶故事，生成 Playwright E2E 測試：

作為學生，
我想要能夠搜尋和報名課程，
以便學習新技能。

驗收標準：
1. 能夠在搜尋框輸入關鍵字
2. 搜尋結果正確顯示相關課程
3. 點擊課程可以查看詳情
4. 能夠點擊報名按鈕
5. 報名後顯示在我的課程列表

請生成完整的測試程式碼，包含正向和負向測試案例。
```
